Using the CLI

All the boards have a command line interface, accessible over UART. To connect to it, use the st-link programming board as you normally would to program. To view the CLI, you can use CoolTerm or Picocom.

CoolTerm

Download CoolTerm here: https://freeware.the-meiers.org). The settings required are:

  • Baud Rate
    • F0 board: 57600 
    • F7 board: 230400
  • Turn on Local Echo
  • Enable tab separated text with separator 16 (optional)
  • Handle BS and Del characters

Note: If you are using a Mac, you may have to use ctrl+J to send commands to your board

Picocom

Picocom is a minimal dumb-terminal emulation program. To download it, run:

sudo apt-get install picocom

An example of opening a picocom session would be:

picocom /dev/ttyACM0 -b 57600 --echo --omap crcrlf --imap lfcrlf --emap lfcrlf

This does the following:

  • Connect to device /dev/ttyACM0 (your device may be something else, e.g. /dev/ttyUSB0)
  • Set baud rate to 57600
  • Turn on local echo
  • Translate `\r` to `\r\n` before being written to serial port (--omap)
  • Translate `\n` to `\r\n` after being read from serial port (–imap)
  • Translate `\n` with `\r\n` before being locally echoed to terminal (–emap)

When running on your own device, keep the --echo, --omap, --imap, and --emap settings the same as the example. The only things you may need to change are device (e.g. /dev/ttyUSB0 instead of /dev/ttyACM0) and baud rate (set to 57600 if using F0 board and 230400 if using F7 board).

To exit a picocom session, enter Ctrl+A and then Ctrl+X.

For a full list of possible commands, read the picocom documentation here: https://github.com/npat-efault/picocom/blob/1acf1ddabaf3576b4023c4f6f09c5a3e4b086fb8/picocom.1.md

Adding CLI Commands

The CLI is programmed using FreeRTOS+CLI. The repositories for each board have a something_mock.c file which implements all the CLI commands through the FreeRTOS+CLI API. For more information on the API, read the documentation here on how to work with it: https://www.freertos.org/FreeRTOS-Plus/FreeRTOS_Plus_CLI/FreeRTOS_Plus_Command_Line_Interface.html
