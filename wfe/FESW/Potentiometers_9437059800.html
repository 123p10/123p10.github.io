Supporting Information, Project Details

Simulation:

  1. Implement 2 potentiometers to the micro-controller
  2. Code the two GPIO pins such that:
    - The analogue signal from the potentiometers are converted to digital signals
    - The signals should be compared with each other ensuring equality (Under Coding)
    - In the case where both the digital values are equal, this voltage will be used to speed up or down the blinking of a green LED
    - In the case where both digital values are not equal, no green light will be emitted; instead, a warning will be given to the user in terms of a red LED which blinks at 250 ms to grab attention. (Under Coding)
  3. Draw and design the schematic diagram

Requirements and Considerations

Cost:

  • The cost of the components involved in the project is $2.50 (maximum ~ not including motors and board cost)
    Tools Purchased:
  • 2 Linear B100K Potentiometers (<$1.00 each)
  • Jumper/breakout wires (<20 cents each)

Constraints:

  • Please note: the designed schematic diagram simulates the actual implementation which uses motors; however, due to the lack of motors at the time of project, the actual circuit and coding represent motors through blinking LEDs. The power given to the motor is proportional to the speed of the blinking of the LEDs as the potentiometers turn.
    This will be updated with the implementation of motors as soon as possible.
  • The design and scope of the project is closed upon the use of potentiometers in cars' pedals since this is the main use. Other uses fall as subsidiary cases and are not as vital; hence, they will be added later after the completion of the main use. 
  • The potentiometer type used in simulation will surely be different than in the car; however, the same code will be applicable
