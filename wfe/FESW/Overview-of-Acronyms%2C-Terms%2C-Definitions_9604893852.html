Firmware : Overview of Acronyms, Terms, Definitions

Created by Owen Brake, last modified by Jeffrey Zhen on May 01, 2022

Often times in the code acronyms or niche terms are used. These can be confusing for new FW developers. This is a compiled list of terms which may be used in the firmware along with their definitions.

Teams

Firmware

  • This team
  • Writes all the firmware for all the boards on the car

Electrical

  • Designs and brings up all the PCB/boards on the car
  • Is involved in the electrical design of the car

Tractive

  • This is a mix of the old Accumulator and Powertrain teams
  • They are essentially a design team that encompasses the moving portion of the car as well as the battery design
  • Powertrain
    • Motors
    • Motor Controllers
    • Gear Ratios
    • Cooling
  • Accumulator
    • The entire battery system
      • 7 parallel groups of 70 cells in series
      • Each group of cells in series is made up of 5 groups of 14 cells with each cell running at a voltage of 2.5V (discharged) to 4.2V (charged)Â 
      • The accumulator is broken down into segments (1/5 of the overall battery voltage) which are 7 parallel groups of 14 cells in series

Mechanical

  • Designs the chassis for the car

Operations

  • Manages the team
    • JIRA
    • Confluence (this)
    • Competitions
    • Longitudinal team planning
    • Sponsorships

Terms used for Team Management

FW - Firmware

WFE - Waterloo Formula Electric

Technical Terms

HV - High Voltage

LV - Low Voltage

GLV/GLVMS - Grounded Low Voltage (Master Switch)

  • A Voltage which is grounded to "earth" as in a wire attached to the chassis
  • GLVMS is switch which shutoffs all power to GLV

EM - Electric Motor

  • It is the button pressed after HV initialized
  • Essentially ready to drive.

CAN - Control Area Network

  • Protocol with which all the boards interact with each other through
  • See documentation for more details how this works

DTC - Diagnostic Trouble Code

  • DTCs are special messages also sent over the CAN bus
  • They specify specific fault conditions
  • See documentation for more details

FreeRTOS

  • This is the operating system used on all boards
  • It is an RTOS(Real Time Operating System), you should research this and have an understanding of how this works

ADC - Analog To Digital Converter

  • These are systems which convert an analog signal like a voltage reading from a thermistor into a readable digital signal
  • They remove noise from signals for a nice digital reading.

HAL - Hardware Abstraction Layer/Library

  • An API or set of functions used in the FW to interact directly with the HW or Processor.
  • One example would be to interface with the SPI interface or the GPIO pins

IL/HVIL - (High Voltage) Interlock Loop

  • An interlock loop are essentially two coupled pieces which if one fails the other fails
  • We have a HVIL which monitors all the HV connections using a LV system. If the LV loop is broken it shuts off the HV loop to prevent potential safety hazards

HVD - High Voltage Disconnect

  • Connector which can easily be activated to disconnect HV on the accumulator

BSPD - Brake System Plausibility Device

  • Non-programmable device which checks if driver is braking and high voltage is being sent to motor
  • We shutdown HV as there is most likely a large issue if the motor is spinning while driver is braking.

TSAL - Tractive System Active Light

  • LED which presents status of HV(tractive) power

AILED - Accumulator Indicator LED

  • LED which presents status of HV power in the accumulator/battery

ESF - Electrical System Form

  • Complete documentation of entire electrical system
  • Submitted in competitions

SES - Structural Equivalency Spreadsheet

  • Documentation on chassis, mechanical structure
  • Submitted in competitions

TC - Traction Control

TV - Torque Vectoring

Precharge

  • A series of operations and tests the BMU performs to apply high voltage to the vehicle bus.
    • Applies voltage from the battery to the high voltage bus
  • There are a series of tests performed to validate that high voltage is being applied properly and safely.
  • The precharge process is used to gradually apply voltage to prevent inrush current potentially causing safety hazards or damage to components.

Boards and Hardware

BMU - Battery Management Unit

  • This is the unit that manages the battery
  • It has 6 (as of May 10, 2021) AMS board attached to it
  • Functions:
    • Interface with AMS
      • Measures Cell Voltage
      • Balance Cells
      • Measure Cell temps through Thermistors
    • React to potential fault conditions in Accumulator/Battery
    • Manage Charging of Battery
    • Manage Battery Cooling

AMS - Accumulator Management System

  • The AMS is essentially the interface that the BMU has with the battery
  • There are multiple (6 as of May 10, 2021) AMS boards on the system
  • These boards are all connected to the BMU
  • They act as peripherals/measurement devices
  • Each board contains an LTC68XX chip which manages, monitors and balances voltage cells as well as the thermistors in the battery
  • No actual WFE Firmware lives on the AMS, they are proprietary chips which we interface using isoSPI

VCU - Vehicle Control Unit

  • Essentially controls driving of car
  • Converting driver input into vehicle output.
  • Controls beaglebone

DCU - Dashboard Control Unit

  • Controls vehicle dashboard.
  • Processes button presses and LED indicators

PDU - Power Distribution Unit

  • Manages LV part of car:
    • Selecting LV source
    • Sending LV power to different parts of car.

CC - Charge Cart

  • This is a board which manages the charging of the accumulator
  • It is off of the car and only used when charging the car

WSB - Wheel Sensor(or Speed) Board

  • This board measures various sensors around the wheels of the car like:
    • Wheel speed
    • Tire Temperature
    • Coolant pressure
  • It currently is a low priority board

COMMON-ALL

  • Is a repository on Bitbucket
  • Is a git submodule included in all major board FW
  • Contains code which is used in all board code like CAN driver, State Machine, etc.

LTC68XX

  • These are the chips which are installed on all the AMS boards
  • The one we currently use (as of May 10, 2021) is LTC6804
  • The LTC6804 is directly addressable so each message sent over the isoSPI interface contains the chip address
  • The LTC6812 uses the isoSPI interface like a shift register, broadcasting messages
  • See documentation for greater details
  • The LTC68XX chips:
    • Report Cell Voltages
    • Report Temperature from Thermistors
    • Discharge/Charge Cells

LTC4110

  • This is our backup energy system manager
  • When HV is disconnected this chip switches over to the LV battery
  • It's managed by the PDU

F0/F7/Nucleo

  • These are all chipsets which we use on our boards
  • F0 is short form for the STM32F0 chipset we use on our boards same for F7
  • Nucleo is a type of dev board which can be used to test code on.

Inverter

  • Inverter is a strange niche term, it basically is a part of the motor controller
  • It essentially converts the DC input into the motor controllers into the AC output that the motor will use for power
  • It's often used as a general term for what you may call a "motor controller"

HITL - Hardware in the Loop

  • The HITL is a hardware project which effectively emulates the function of the car
  • It is a board made up of various potentiometers, sensors and switches which can be used as sort of mock car functions
  • Boards can be connected to the HITL and software can be tested without having to have a full chassis and working peripherals.

ADE79XX

  • The one currently used is the ADE7912
  • It is essentially a HV ADC

BRB - Big Red Button

  • Previously the emergency stop button on the side of the car, new rule change require multiple resettable stop buttons around the car
