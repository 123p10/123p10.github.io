Summary

Bring up new Car computer so teams can work cross functionally. This involves getting the software together. Making a decision on the OS. It also involves specing out the hardware we need and constructing it.

Requirements

Software

  • Git

  • Some sort of CAN software

    • Need to be able to Read and Write CAN messages on the CAN bus

    • Should be able to graph in some way

    • Should be able to read and edit DBC files

  • Visualizations

    • Should interface well with our existing visualizations (if we have any)

  • Python

  • TBD

    • Before making any decisions we should consult other teams.

UX

  • The Software UX needs to be intuitive so other non firmware members can use it.

    • CLI should also be supported so that firmware members can make changes on git and pull new firmware.

  • Should be intuitive to use so keyboard + mouse

Hardware

  • Good large display

  • Ability to connect to CAN bus

  • Fast, long lasting hardware

    • Preferably upgradeable so we don’t always have to redesign every X years

  • Should be designed so that it can fit easily in the bay.

  • Would be nice if it was somewhat portable, as in brought to competition

  • Networking

Software

OS

  • The OS selection is tough. It would be really nice to use Linux because that is completely compatible with all our firmware systems. But we have to have good CAN software and we need to consider the needs of other teams.

    • On Linux I’ve found and used SavvyCAN a bit, I’m sure there are others but this should be investigated before we settle on Linux

Use Cases

  • The number one use case is so that other teams can read CAN messages off the car when they are testing various subsystems.

  • It would be nice if other teams could flash new firmware from the computer as well. This is connected to the “Flash FW over CAN“ project

  • NOTE: Other teams will have use cases so they should be consulted

  • It would be nice if we could run in the background a jenkins server

  • Being able to process and analyze statistics on here would be nice so collecting our telemetry straight off the CAN bus and visualizing it.

Process

There are 4 steps to solving this issue and they should go in this order:

  1. Spec out all the software ensure it will all work with the hardware

  2. Compile a list of all the parts needed to construct the computer

  3. Reach out for funding

  4. Order and construct bay computer
