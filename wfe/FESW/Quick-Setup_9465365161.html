Install Guide (Old)

Note: This guide is outdated. Please follow the Vagrant setup above if you haven't set up a VM yet.

  1. Virtual Box, STM32CUBEMX and the VM files are located here
  2. You might need to install winrar (or The Unarchiver for Mac) to decompress Linux_Mint.zip from the VM Environment folder
  3. Install Virtual Box
  4. In Virtual Box go to machine → add and select the .vdi file that you unzipped
  5. Launch the virtual machine
    1. Note: the password is: firmware
  6. On boot, open (double click on) home/Desktop/run-me-on-startup.sh and run via terminal to create/mount a case-insensitive filesystem
    1. Linux is a case-sensitive OS.  All work must be done on a case-insensitive filesystem that is mounted by the step above (files should persist in this mount between reboots)
    2. The purpose of this is because the code generated in STM32CUBEMX uses case insensitive file directories and linux won't be able to find them
  7. When first setting up the VM for yourself please create a new SSH key and link with your Bitbucket account
  8. You may want to install guest additions, to allow shared clipboard, shared folder, and better graphics support. If so, follow this link. If you get an error inserting guest additions drive, do as described in the error message by going to storage settings, and beside the SATA controller options select add disk and add an empty disk
  9. Mount a shared folder through Virtual Box.  If this does not work then follow this guide
    1. Make sure to enable Auto Mount when creating the shared folder.
  10. To make sure everything was installed correctly follow the Blinking LEDs tutorial
