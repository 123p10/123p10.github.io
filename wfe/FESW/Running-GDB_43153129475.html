Firmware : Running GDB

Created by Justin Vuong, last modified by User 6fab7 on Sep 13, 2022

Below outlines the steps needed to run GDB for each of the boards.

xxx represents the name of a board (ex. pdu, bmu, vcu)

  1. Make sure you are in the right directory

    cd firmware

  2. Run command to flash code, you must specify the target board. You must also enable debug mode.

    make load LOAD_TARGET=xxx DEBUG=1

  3. Run the connect command 

    make connect LOAD_TARGET=xxx DEBUG=1

  4. Wait for this line to appear in the terminal

  5. In a SEPERATE terminal different from the one used for the previous step: 

    make gdb LOAD_TARGET=xxx DEBUG=1

*Note: Make sure the load target is the same throughout
