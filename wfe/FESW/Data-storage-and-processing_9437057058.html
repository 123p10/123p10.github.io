Progress Outline and Documentation

Apache Cassandra

  • The Missouri S&T Solar Car Team published an article about their research into and testing of possible database solutions for their telemetry needs.
    • Executive Summary:
      • Due to the nature of telemetry data collection, they needed a system that could both read and write to a database extremely quickly, no matter how large the tables become.
      • They conducted tests against MySQL, an SQL database system, and Apache Cassandra, a NoSQL database designed for scalability and high-availability.
      • Their tests consisted of inserting a large number of rows and tracking the query for each operation as data grows larger.
      • Their results showed that as you return more and more rows from MySQL, each query grows slower.
      • Conversely, their results from Apache Cassandra showed that query times were nearly stable as the number of rows increased.
      • Furthermore, Cassandra's write times were roughly 97% faster than MySQL's.
      • Unsurprisingly, they opted to use Apache Cassandra for their telemetry needs.
    • Article: http://solarcar.mst.edu/news/725/choosing-a-database-for-solar-car-telemetry-data
    • Apache Cassandra: http://cassandra.apache.org/
      • Apache Cassandra is completely open-source.
      • Used by Apple, Netflix and Ebay.
      • ThingsBoard's (telemetry framework) recommended DB option.
