Connecting to the Beaglebone

This is based on the assumption that the SD card has the correct OS installed. If you experience issues, it might be because the SD card needs to be flashed with the proper OS (2020_beaglebone_os in bitbucket)

via SSH

  1. On your host machine (your laptop) connect to eduroam (guide)

    1. You may actually have to connect to the Beaglebone’s network with credentials:

      1. SSID: WFE-XXXX

      2. password: firmware

  2. Connect to beaglebone black via SSH

    1. If you prefer to use your own SSH client that’s fine. The settings should be something like:

      1. TODO: The below 3 settings will probably change in an upcoming issue, so hopefully who ever makes that change updates this page

      2. IP: 192.168.7.2

      3. username: debian

      4. password: temppwd

    2. If you just want to use the command line then just run:

      1. ssh debian@192.168.7.2

      2. In the password prompt type: “temppwd“

Running Software

  1. The main software should be in “/home/debian/2020_beaglebone_app“ a.k.a “~/debian/2020_beaglebone_app”

    1. If it is not then goto the section labelled “Setting up 2020_beaglebone_app“

  2. Run “pip3 install .“ inside 2020_beaglebone_app

    1. This is currently broken as PySide2 as a dependency was added to the 2020_beaglebone_app package even though there is no pip package for ARM

      1. The solution I did was run “sudo apt-get install python3-pyside2.qtgui“

      2. Then remove “PySide2“ from requirements.txt and setup.py

Setting up 2020_beaglebone_app

Unfortunately, we don’t have a beaglebone Bitbucket account, nor have we setup ssh keys for the beaglebone. So we have to manually copy 2020_beaglebone_app to the beaglebone.

  1. Open up a new terminal on your host machine (laptop)

  2. clone 2020_beaglebone_app in some folder (now known as <path_to_app>)

    1. This can be done easily by navigating to some folder and running:

      1. git clone git@bitbucket.org:waterloohybrid/2020_beaglebone_app.git

  3. Init the common-all submodule

    1. Run: “git submodule init“

    2. Then: “git submodule update --init --recursive”

  4. Then perform an scp of the folder <path_to_app> into the beaglebone.

    1. This is done by running this command with <path_to_app> as some directory “/…/<myusername>/…/2020_beaglebone_app/“

    2. scp -r <path_to_app> debian@192.168.7.2:/home/debian/2020_beaglebone_app

  5. The folder /home/debian/2020_beaglebone_app should now be filled out
