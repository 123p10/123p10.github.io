October 6, 2021

Priorities:

  1. 2021 Vehicle Testing / Getting 2022 Car working
    1. This involves bringing up new 2022 AMS board
    2. Running the various tests as shown on the subpages of this page Car Testing Procedure
    3. Pushing the 2021 car to it's limits to attempt to identify and resolve any bugs
    4. Members:
      1. Ryan Erler is heading this up right now in collaboration with tractive and electrical members
  2. Logging, Beaglebone, Heads up Display
    1. We need to improve CAN logging
      1. This functionality should already exist
      2. We need to ensure that it's working as expected
      3. We need to create the proper hooks to make it easy to view the CAN logs
    2. We need to improve our general logging
      1. This includes things like IMU/GPS (though these are less of a priority)
    3. We need to make more use of the heads up display
      1. Display DTCs
      2. Attempt to make human readable errors so the driver can react to them
    4. These features will help make sure our car is functioning properly and that non firmware members can detect and resolve car issues
      1. This can also improve our scores with judges
    5. Members:
      1. Zulaikha is heading this up now in particular CAN logging
  3. Endurance Mode
    1. This is a feature that we are implementing to improve our car's score
    2. Endurance mode is a feature to attempt to get the vehicle to travel as far as possible
    3. There is an endurance event at our competitions
    4. The process for endurance mode is generally:
      1. Compute Battery SoC
      2. Compute distance travelled and rate of discharge
      3. Restrict/Inform driver on speed to ensure completion of the race
    5. Members:
      1. This is Justin's current task
  4. Traction Control, Torque Vectoring
    1. These are 2 features which would help improve vehicle performance
    2. They are low priority until the above features are complete
  5. WSB
    1. Low priority
    2. Goal right now is just get them on the car
    3. They fill very little actual function
  6. Quality of Life (monorepo, linting, unit testing)
    1. These are things we'd like to have but are not critical to car performance
    2. Should be given to new members

Extra Notes

  • We need to buy new beaglebones ASAP
  • CAN bootloader is on the backburner until higher priority issues are resolved
  • We are prioritizing risk management over feature completion
