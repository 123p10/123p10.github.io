Firmware : How to find driver requirements for a board

Created by Former user, last modified on May 17, 2018

This article will explain how to read a schematic to determine the driver requirements for boards.


Assumptions:

  1. Latest board files have been cloned from Bitbucket
  2. User understands how to check file status, navigate, and pull from Bitbuckets
  3. Appropriate branch has been cloned - depending on the board file needed

Step-by-step guide

Once you understand all the assumptions below the following procedure can be followed.

  1. Check out the appropriate branch, ie under 2018_designs → BMU
  2. Open the .PrjPcb, this open Altium with all the files for the project.
  3. Within Altium, open the Proc.SchDoc that contains the layout of the MCU for the board.
  4. The MCU layout can be read as follows:
    1. The label PA0 stands for "Port A" "Pin 0" - the same pattern for all the others
    2. The yellow tags are called "Ports" and refer to higher level inputs, outputs, or bidirectional
      1. If arrow is going to the Pin - its an input ie "CONT_NEG_SENSE"
      2. If arrow is going out of Pin - its an output  "CONT_NEG
      3. Blue box refers to communication buses

All the drivers on the firmware are direct nets going into the MCU.
