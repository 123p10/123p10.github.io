Firmware : Feature Design Specifications Guide

Created by Owen Brake, last modified on Oct 09, 2021

In these subpages will be a list of various design specifications applicable to firmware. The design spec should include the following:

  1. Status of the Project

    1. Unstarted

    2. Development in Progress

    3. Review

    4. Complete

  2. An attached Jira ticket

  3. Spec author/owner

  4. Summary of purpose for this feature/design

  5. Description of how this will be or is implemented.

Use figures as much as possible.
