Firmware : Vector CANAnalyzer and Vector CANdb++

Created by ba2nguye@uwaterloo.ca, last modified by Justin Vuong on Jun 16, 2022

Vector CANdb++ is used to define the car's CAN network. It is used to add new CAN messages, signals, etc.

Vector CANAnalyzer is used along with the Vector CANAnalyzer hardware dongle to view in realtime messages being sent on the car's CAN bus.

 Vector CANdb++ comes bundled with Vector CANAnalyzer, so install Vector CANAnalyzer to use either.

Step-by-step Install guide

  1. Download all required files:
    1. Car cand.dbc file - this is a database file that contains all CAN messages for the vehicle. For the 2018-2020 car, we use the 2018CAR.dbc file available from the common-all Bitbucket repository here under the Data folder
    2. CANAnalyzer installer (make sure it corresponds to the license) (alternative download)
    3. Vector USB Driver
  2. Installation
    1. Follow the CANAnalyzer install
    2. To verify if the USB driver was correctly installed, go to "Device Manager" plug the Vector tool and locate "Vector Hardware"
    3. In order for the functions to be fully unlocked, the CAN tool must be connected which has software installed on it

Software basics

Quick start guide: CANalyzer_QuickStart.pdf

CANdb++

To access CANdb++, open CANAlyzer, go to tools→CANdb++ editor

CANAlyzer

    1. The software is capable of sending and receiving messages and other high level analysis of the CAN bus
    2. Configuration bar:
      1. Input an "Interactive Generator" to generate and send CAN messages from the vector tool
        1. Different CAN standards are supported and must be selected. After adding a message, make sure to check a box next to time interval to actually send the message
      2. Add "database" i.e. .dbc file for the software to automatically identify and break down CAN messages or use them as templates
      3. Start/Stop to actually start sending/receiving messages
    3. Trace bar:
      1. By checking this bar, you can see all messages in Rx and Tx (receive/transmit) directions
      2. Can also identify errors in the CAN bus
        1. A common one is "NACK filter" which really means the messages transmitted to a board are not being received properly
