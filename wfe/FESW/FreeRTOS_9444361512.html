FreeRTOS and Interrupts

FreeRTOS uses interrupts extensively for context switching and other tasks. If your code uses interrupts, make sure to set the interrupt priority to co-exist with freeRTOS APIs if necessary (https://www.freertos.org/a00110.html#kernel_priority). If you use STM32Cube, it should set this correctly for you automatically

Also note that between any freeRTOS API call, and when the scheduler is started, interrupts won't work (not entirely sure what happens, but they don't seem to behave as expected). So try to avoid putting any code in between those two functions.
