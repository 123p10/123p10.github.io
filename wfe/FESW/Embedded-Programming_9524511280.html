Firmware : Embedded Programming

Created by Former user on Sep 13, 2019

The main thing that differentiates embedded programming from normal programming, at least to a beginner, is controlling hardware. This is done through device drivers, which provide an interface to control the hardware. For our boards, we use the STM32 Hardware Abstraction Layer (HAL), which is a collection of device drivers for the micro controller we use. The HAL has some documentation available at the following link: 

https://www.st.com/content/ccc/resource/technical/document/user_manual/2f/77/25/0f/5c/38/48/80/DM00122015.pdf/files/DM00122015.pdf/jcr:content/translations/en.DM00122015.pdf


For more information on device drivers, see the following tutorial on device driver development. You won't necessarily need to implement the code it describes, however understanding the general concepts is important.

 https://blog.atollic.com/device-driver-development-the-ultimate-guide-for-embedded-system-developers
