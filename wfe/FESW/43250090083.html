Firmware : How to pull/reset CAN logs

Created by Justin Vuong on Oct 06, 2022

If you are on a Windows machine download: git bash

If you are on linux/mac open a terminal window

Connect to the beaglebone’s wifi

SSID: WFE-6C60
password: firmware

How to view CAN logs

Ssh into the BB

#Your local computer terminal
ssh debian@192.168.7.2 # password is "tmppwd" no quotation marks

All can logs are in the directory ~/can_monitor_logs

How to pull logs

scp the directory to your local computer

#Your local computer terminal
scp -r debian@192.168.7.2:~/can_monitor_logs .

Now move the files to a new directory. ssh into the BB

  • MAKE SURE THAT YOU COORDINATE THE <folder_name> USED IN THIS PART WITH THE PERSON TAKING NOTES 

#Your local computer terminal
ssh debian@192.168.7.2 # password is "tmppwd" no quotation marks
#On the beaglebone terminal
cd can_monitor_logs
mkdir <folder_name>
mv *.log <folder_name>

Validate that the logs are fine

Navigate to your firmware repo in the command line. We want to convert the candump output to csv’s that we can read and process

#Your local computer terminal
cd beaglebone/app/wfe/post_processing/
#Replace AugXXXXXX.csv with your filename and make sure its identifiable for the note taker
python3 parse_candump.py candump_log.log > AugXXXXXX.csv
python parse_log.py -r "CurrentBusHV\\d" graph -ymin 0 -ymax 400 AugXXXXXX.csv
python parse_log.py -r "VoltageCell0\\d" graph -ymin 0 -ymax 4.2 AugXXXXXX.csv
