Firmware : GPS Documentation

Created by Justin Vuong, last modified by Alexander Cholmsky on Aug 15, 2022

Getting a fix

The frequency with which the red FIX light blinks indicates if a fix has been found:

  1. If the LED is blinking once a second, then it is searching for a fix.

  2. If the sensor has a fix, the LED will blink once every 15 seconds.

In order to get a 3D fix, (with time, latitude, longitude and altitude) a connection must be established with at least four satellites. This is what the Adafruit Ultimate GPS Breakout v3 will look for.

  • Note that if precise time is not required, only three satellites are needed.

Tips to get an initial fix:

  • Bring the sensor outside into a large, open area

  • Wait for the frequency of the FIX light to change from blinking once a second to once every 15 seconds.

Interpreting the output

Sample output:

In order to convert the output to (X,Y,Z) coordinates, you can use the following equations.

Ellipsoidal model

Radius: r = 6378137 + alt 

x = r*cos(lat)*cos(lon) 

y = r*cos(lat)*sin(lon) 

z = r*sin(lat)(1 – 1/298.257223563)

Code overview

  1. On Init, we use ser.write(“string”) to send the GPS sensor the update rate we want and the type of messages we want

    1. Currently set to 1 Hz and latitude and longitude

  2. We then call the read() function, which waits for serial output from the GPS sensor and splits this output into strings with our desired data (latitude and longitude)

  3. This data is then written to a CSV file

Notes from testing:

Running multiple processes using cron is not a good idea. Cron has no knowledge of resource availability, thus if a process requires a network that may not be available yet the process will simply fail without errors. The same process may run successfully when being run through the command line, but not @reboot. Should use systemd for multiple processes as it provides more flexibility in this respect.
