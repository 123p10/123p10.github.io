Description

HVADC (analog devices) measure voltage and current coming out of the battery.  Tyler wrote this driver (should be documented).

Using driver in firmware code:

  • Include the header file
  • Make sure SPI communication is setup with the ADC and initialized.
  • Call the desired function.

Procedure for calibrating HVADC driver:

  • Have matlab and the visa driver installed.
  • Run code on the bmu that waits on UART communication then responds with the raw value from the ADC (see while loop below).
  • Open up the HV_ADC.m matlab script.
  • Go to Apps > Instrument Control > VISA > USB
  • Connect to the instrument that you are using to test.
  • Copy the connection code into the "Instrument Connection" section of the matlab script.
  • Run the script to generate data.
  • Use the plot() function from the command window to generate a graph of the relationship between voltage and ADC value.
    • Ex: plot(data(:,2), data(:,1))
  • From the plot window go to Tools > Basic Fitting then check the linear and Show equations checkboxes.
    • This will show the best linear fit with the equation to convert from raw value to actual voltage.
    • The plot residuals checkbox can also be used to view the error of the linear fit.
  • Replicate the equations in the driver code.
    • I used two separate functions in my code since it seems like the negative and positive voltages have a slightly different relationship to the raw values.

Example code for bmu to output raw values used in calibration:

while (1)
{
	HAL_UART_Receive(&huart3, dataUART, lengthUART, 10000000); //Block until something is received over UART
	int32_t rawValue = ADC_Read_Value(&hspi1, HV_ADC_CS_GPIO_Port, HV_ADC_CS_Pin, 0b00001100); //Get the raw V1 value from the ADC
	printf("%ld\r\n", rawValue); //Send the value through UART
}
