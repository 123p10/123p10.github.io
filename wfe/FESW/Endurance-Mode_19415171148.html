Summary

Method to maximize vehicle distance and complete endurance competition (44km) in most efficient manner.

Requirements

UX

  • Driver should be able to toggle into and out of endurance mode

  • Driver should be able to abort endurance mode anytime during driving

  • Driver should have some indication of whether they are in endurance mode

Vehicle Performance

  • Vehicle must be able to complete endurance distance

  • Vehicle should not be excessively slow so as to achieve a good time and efficiency in endurance competition

Design - UX

Design UX - Button Trigger

The TC (Traction Control) and TV (Torque Vectoring) buttons are not used on the current 2021 car dashboard. They could be used to toggle in and out of endurance mode.

Design UX - Dashboard

The driver needs a way to know when they’re in endurance mode. It would probably be smart to have an indication on the dashboard that indicates to the driver they’re in endurance mode.

It would also be a very nice thing to add a State of Charge Indicator on the dashboard. This should be the bare minimum for endurance mode. Basically provide State of Charge info to the driver and let them manage the car’s speed along the track.

Design - Vehicle Performance

Design Vehicle Performance - Driver Managed

Essentially, just let the driver look at the state of charge indicator and make a judgement on how fast to go to finish the race.

Pros

Cons

Dead simple

Relies on driver to have experience and incredible skill

Unobtrusive, very low chance it will cause the driver to mess up

Not rigorous, probably not the best method we can use

Design Vehicle Performance - Feedback Controller

Description

This is what is sort of expected for “Endurance mode“.

Essentially we need to design a feedback controller which modifies the throttle pedal map based on the remaining range. I’ve designed a feedback controller which I think could apply to our use case. The diagram shown below is a PID controller which uses the expected endurance competition distance and the current remaining vehicle range as the error variable. Then the error is passed through our PID controller and is used as feedback into our motor throttle algorithm.

I’ve attached some PID controller documentation in the Appendix.

As a summary of the above chart.

  1. Determine how much the vehicle has currently travelled

  2. Determine how much range the vehicle has left

  3. Subtract the remaining range from the endurance competition to get the error value

  4. If the error value is < 0, then the driver will finish the course so perform no corrections, else continue

  5. Plug the error into the PID controller to get the throttle feedback

  6. Clip the feedback at some maximum value, we don’t want the endurance mode calculation to stop the vehicle

The PID equation is shown simply below, where E is the error value (Competition Distance - Vehicle Range).

This model is a bit complex but it would provide us with a robust, accurate feedback loop to ensure we go the maximum distance in the maximum speed.

Potential Issues

There are some roadblocks as detailed below.

The first one is PID tuning. The Kp and Ki values are constants which must be tuned. I’ve only tuned PID constants on rapid motion devices. This PID controller would be on a system where the Kp, Ki values would have to be tested over the entire range >40km. These values could be tested on a dyno (dynamometer), but I am unsure of our access to one, and it would still take hours in between tuning because the vehicle has to traverse and then recharge the battery.

The second one is the potential for intrusion on driver behaviour. We really don’t want to have our endurance mode negatively affect our performance due to some programming error. So to prevent this there will be 2 solutions: 1 is there will be a cap on the feedback loop effect so the loop cannot force the throttle to 0 or some arbitrary low value, 2 there will be an endurance mode abort button which turns off this feedback loop and just lets the driver drive.

Pros

Cons

Accurate

Tuning PID controller will be difficult

Would provide the most correct “endurance mode“, with truly, mathematically determining the fastest long distance run

Very complex

Very cool to show judges

Potentially intrusive

  • Though we will have an abort button

Appendix

Note, there are some nuances here which haven’t been discussed. First, it’s assumed that the faster you drive the more energy is expended but this isn’t true. We should work with tractive on this.

As shown in the below figure, maintaining a speed of 40mph is actually the most optimal speed for a Model X, driving slower than that will result in lower range.

We should maybe perform some dyno testing for this.
