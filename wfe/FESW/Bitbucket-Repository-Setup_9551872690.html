Firmware : Bitbucket Repository Setup

Created by Former user, last modified on Jan 25, 2020

Here are some important tasks to do when creating a new Bitbucket repository. Some of them may require admin permissions on Bitbucket

  1. Make private repository
  2. Set-up issue tracker
  3. Create readme (ideally with a link to a confluence page for more information)
  4. Create new issue message
  5. Create branches according to gitflow strategy: https://www.atlassian.com/git/tutorials/comparing-workflows/gitflow-workflow
  6. Set up branch permissions so master and develop are protected, only allow merge via pull request and require approval from at least 1 default reviewer
  7. Create default pull request message template
  8. Create default reviewers for pull requests
  9. Default merge strategy should be set to merge commit
